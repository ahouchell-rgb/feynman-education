## Files
- `<UNIT>.jsonl` ŌĆö one file per app unit (e.g. `C1.jsonl`), IDs `C1-0001ŌĆ”`.
- `_all.jsonl` ŌĆö every question, concatenated.
- `_manifest.json` ŌĆö per-unit metadata: title, discipline, source handbook + printed
  page ranges, subtopics, and the target counts used during generation.

Each line is one minified JSON object. The `unit` code matches the Springboard course
units in `apps/houchell/public/learn/content.js` (which seed mastery-graph qids), so
the bank can be tied to the mastery graph by unit/objective.

## Schema (one JSON object per line)

```jsonc
{
  "id":         "C1-0007",                 // <unit>-<seq>
  "unit":       "C1",                       // app unit code
  "topic":      "The particle model",       // unit title
  "subtopic":   "Gas pressure",             // handbook subtopic
  "discipline": "chemistry",                // biology | chemistry | physics | earth
  "skill":      "Explain",                  // Recall|Define|Describe|Explain|Apply|Calculate|Compare|Interpret
  "difficulty": "Core",                     // Foundation | Core | Higher | GCSE-foundation
  "type":       "short",                    // short | mcq | numeric | multi
  "marks":      2,
  "question":   "Why does gas pressure increase when a sealed can is heated?",
  "answer":     ["particles move faster", "hit the walls more often", "and with more force", "so pressure rises"],
  "accept":     ["faster particles", "more frequent collisions"],
  "misconceptions": ["the particles expand", "the particles get heavier", "heat rises so pressure rises"]
}
```

Type-specific extra fields:
- **mcq** ŌĆö `options` (3ŌĆō4), `correct` (0-based index), `distractors` (one rationale per wrong option).
- **numeric** ŌĆö `numeric_answer`, `unit_of_measure`, `working` (worked solution).
- **multi** ŌĆö `options`, `correct` (array of indices).

## Provenance & method
- Handbooks extracted to text (`pypdf`), sliced to each unit's printed page range
  (offset +2, contents-verified). Each of the 35 app units maps onto the handbooks'
  27 units / ~96 subtopics (e.g. `PWR` = handbook P1.3; `MIC` = B1.1 + B1.4;
  `PRS` = C1.4 gas pressure + P4 pressure in fluids).
- Generated by 66 parallel agents (units >40 questions sharded by subtopic, Ōēż40/agent),
  each grounded in the verbatim handbook text plus the unit's distilled facts.
- Post-processed: schema-validated, `accept`/`misconceptions` normalised to arrays,
  deduplicated within each unit (54 near-duplicates dropped), sequential IDs assigned.

Regeneration scripts live in the session scratchpad (`build_grounding.py`,
`qbank_workflow.js`, `assemble.py`); the grounding excerpts are derived from
copyrighted handbooks and are **not** committed ŌĆö only the generated questions are.
